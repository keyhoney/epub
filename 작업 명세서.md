# 📝 [작업 명세서] Next.js 기반 무설정 ePub 3.0 온라인 에디터 (WYSIWYG)

Next.js를 활용하여 사용자가 코딩이나 ePub 구조를 전혀 몰라도 워드 프로세서처럼 직관적으로 전자책을 만들 수 있는 웹 에디터 서비스의 **바이브 코딩(Vibe Coding)용 프롬프트 겸 작업 명세서**입니다.

이 명세서를 AI(Cursor, Claude 등)에게 단계별로 입력하여 코드를 생성·리팩터링하세요. 모든 연산 및 파일 압축은 **서버(API) 없이 브라우저(Client-side)** 에서 처리합니다.

## 0. 작업 방향 결정 (기존 코드 vs 명세서 통합)

### 채택 방향: **WYSIWYG 우선 리팩터링**

| 구분 | 결정 | 이유 |
|------|------|------|
| 편집 방식 | **Tiptap WYSIWYG** 채택 | 비개발자가 XHTML/CSS를 직접 다루는 것보다 워드 프로세서형 UX가 목표에 부합 |
| 데이터 모델 | **`BookState` / `Chapter`** 채택 | 파일 탐색기(`FileNode[]`) 모델은 개발자용 — 일반 사용자에게 불필요한 인지 부담 |
| ePub 패키징 | **명세서 규격**으로 재구현 | 기존 export는 mimetype 미압축·폴더 경로 불일치 등 규격 미준수 |
| 기존 `app/page.tsx` | **전면 교체** | PubForge 코드 에디터 UI/로직을 WYSIWYG 아키텍처로 대체 |

### 기존 코드에서 **유지·흡수**할 UX (더 나은 경험)

| 기능 | 처리 |
|------|------|
| `localStorage` 자동 저장 | ✅ 유지 — `BookState` 직렬화 후 복구 |
| iframe 실시간 미리보기 | ✅ 유지 — 선택된 장 + 테마 CSS 반영 |
| 읽기 테마 프리셋 (Classic Sepia, Midnight 등) | ✅ 유지 — 사이드바 **「읽기 테마」** 로 단순화 (CSS 코드 편집 UI 제거) |
| 토스트 알림 (`motion` + lucide) | ✅ 유지 |
| 3단 분할 레이아웃 | ✅ 유지 |
| 장 추가·삭제 | ✅ 유지 — 파일명 개념 대신 **장 제목** 중심 |

### 기존 코드에서 **제거**할 항목

| 기능 | 처리 |
|------|------|
| XHTML/CSS `<textarea>` 코드 에디터 | ❌ 제거 |
| Project Explorer (nav.xhtml, styles.css 파일 트리) | ❌ 제거 — 백엔드에서 자동 생성 |
| 드래그앤드롭 코드 템플릿 삽입 | ❌ 제거 → Tiptap 툴바 **「블록 삽입」** (인용구, 정보 박스 등)으로 대체 |
| 가짜 Share 모달 (고정 URL) | ❌ 제거 |
| HTML 단일 파일 Export | ❌ 제거 (핵심 가치는 `.epub` 다운로드) |
| `@google/genai` / 서버 API 연동 | ❌ 사용 안 함 |

### 현재 프로젝트 환경 (이미 준비됨)

- Next.js 15 (App Router), React 19, Tailwind CSS 4, TypeScript
- `jszip`, `lucide-react`, `motion`, `clsx`, `tailwind-merge` 설치됨
- **추가 설치 필요:** `@tiptap/react`, `@tiptap/starter-kit`, `@tiptap/extension-image`, `@tiptap/extension-text-align`, `@tiptap/extension-link`, `@tiptap/extension-placeholder`, `@tiptap/extension-underline`, `@dnd-kit/core`, `@dnd-kit/sortable` (장 순서 DnD)

## 1. 프로젝트 개요 & 기술 스택

**목표:** 복잡한 ePub 3.0 폴더 구조, XML 메타데이터, HTML5 서식을 몰라도 **제목·본문 입력만**으로 유통 가능한 표준 `.epub` 파일을 생성하는 웹 서비스.

**기술 스택**

- Next.js (App Router), Tailwind CSS, TypeScript
- UI: lucide-react, motion (기존), Tailwind Typography (`@tailwindcss/typography` — 미리보기용)

**핵심 라이브러리 (Client-side 전용)**

| 용도 | 라이브러리 |
|------|-----------|
| 리치 텍스트 에디터 | `@tiptap/react`, `@tiptap/starter-kit` + 확장 (image, text-align, link, placeholder) |
| 장 순서 DnD | `@dnd-kit/core`, `@dnd-kit/sortable` |

**권장 폴더 구조**

## 2. 핵심 아키텍처 & 데이터 구조

사용자 입력은 React State(`BookState`)로 관리하며, **다운로드 순간에만** ePub 패키징 모듈(`lib/epub/buildEpub.ts`)로 전달합니다. XHTML·OPF·nav 파일은 사용자에게 노출하지 않습니다.

```typescript
// lib/types/book.ts

export type ReaderThemeId =
  | 'theme-classic'
  | 'theme-midnight'
  | 'theme-modern'
  | 'theme-comfort';

export interface Chapter {
  id: string;           // crypto.randomUUID()
  title: string;
  content: string;      // Tiptap HTML 문자열 (body 내부만)
  order: number;
}

export interface BookMetadata {
  title: string;
  author: string;
  publisher: string;      // 선택, 기본값 빈 문자열
  language: string;     // 기본값 'ko'
  description: string;    // 선택
  coverImage: string | null;  // data URL (표지 업로드)
}

export interface BookState {
  metadata: BookMetadata;
  chapters: Chapter[];
  readerThemeId: ReaderThemeId;  // 기존 테마 갤러리 흡수
}
```

**저장 키:** `localStorage` → `epub_editor_book_state` (기존 `pubforge_files` 키는 마이그레이션 시 무시·삭제)

## 3. UI/UX 구성

단일 페이지, **3단 분할 레이아웃 (Sidebar — Editor — Live Preview)**.

### 헤더

- 앱 이름 (예: **ePub Studio**)

- **[.epub 다운로드]** — 핵심 CTA (헤더 또는 우측 패널 상단)

### 좌측 사이드바 — 책 구조 & 설정

1. **책 정보 폼:** 제목*, 저자*, 출판사, 소개(설명), 표지 이미지 업로드
2. **읽기 테마:** 4종 프리셋 카드 (기존 Classic Sepia / Midnight / Modern Serif / Reading Comfort)
3. **장(Chapter) 목록**
   - [+ 장 추가]
   - 각 항목: 제목 인라인 편집, 선택 하이라이트, **드래그앤드롭 순서 변경**, [삭제]
   - `nav.xhtml`·파일명은 사용자에게 보이지 않음

### 중앙 — WYSIWYG 에디터

- 선택된 장의 본문 편집
- **Tiptap 툴바:** 굵게, 기울임, 밑줄, H2, H3, 정렬(좌·중·우), 링크, 이미지, 인용구(blockquote), 구분선
- **블록 삽입 메뉴** (기존 EPUB 템플릿 대체): 드롭 캡 문단, 정보 박스(aside) — 클릭 한 번으로 삽입
- `onUpdate` 시 해당 `Chapter.content`를 State에 즉시 반영

### 우측 — 실시간 미리보기

- **스마트폰 프레임** UI (둥근 모서리, 상단 노치 또는 상태바 장식)
- `iframe` + `srcDoc`: 선택된 장 HTML + `readerThemeId` CSS + 표지 없이 본문만
- 장 전환 시 미리보기 즉시 갱신
- (선택) 이전/다음 장 네비게이션 버튼

## 4. ePub 3.0 클라이언트 패키징 규칙

다운로드 시 `buildEpub(bookState)`가 아래 ZIP 구조를 생성합니다.

```
my-book.epub
├── mimetype                          # STORE, 비압축, ZIP 내 첫 엔트리
├── META-INF/
│   └── container.xml
└── OEBPS/
    ├── content.opf
    ├── nav.xhtml                     # EPUB 3 표준 nav 문서 (toc.xhtml 아님)
    ├── styles/
    │   └── style.css                 # readerThemeId 기반 CSS
    ├── text/
    │   ├── chapter1.xhtml
    │   └── chapter2.xhtml
    └── images/                       # 본문·표지 이미지 (Base64 → 바이너리 파일)
        ├── cover.jpg
        └── img-001.png
```

### ① mimetype

- 경로: `/mimetype`
- 내용: `application/epub+zip`
- **JSZip 구현 필수:**

```typescript
zip.file('mimetype', 'application/epub+zip', { compression: 'STORE' });
```

### ② META-INF/container.xml (고정)

```xml
<?xml version="1.0" encoding="UTF-8"?>
<container version="1.0" xmlns="urn:oasis:names:tc:opendocument:xmlns:container">
  <rootfiles>
    <rootfile full-path="OEBPS/content.opf" media-type="application/oebps-package+xml"/>
  </rootfiles>
</container>
```

### ③ OEBPS/content.opf (동적 생성)

`BookState` 기반 XML 생성. 필수 요소:

- `<package version="3.0" unique-identifier="pub-id">`
- **metadata:** `dc:title`, `dc:creator`, `dc:language`, `dc:identifier` (`urn:uuid:...`), `dc:publisher`(있을 때), `dc:description`(있을 때), `meta property="dcterms:modified"` (ISO 8601)
- **표지:** `meta name="cover" content="cover-image"` + manifest item `properties="cover-image"`
- **manifest:** 모든 리소스 등록 (`href`는 ZIP 내부 경로와 **정확히 일치**)
  - `nav.xhtml` → `properties="nav"`
  - `text/chapterN.xhtml` → `application/xhtml+xml`
  - `styles/style.css` → `text/css`
  - `images/*` → `image/jpeg` 또는 `image/png`
- **spine:** 본문 chapter만 `<itemref>` (nav 문서는 spine에 넣지 않음)

### ④ OEBPS/nav.xhtml (EPUB 3 목차)

```xml
<?xml version="1.0" encoding="UTF-8"?>
<html xmlns="http://www.w3.org/1999/xhtml" xmlns:epub="http://www.idpf.org/2007/ops">
<head>
  <title>목차</title>
  <link rel="stylesheet" type="text/css" href="styles/style.css"/>
</head>
<body>
  <nav epub:type="toc" id="toc">
    <h1>목차</h1>
    <ol>
      <!-- chapters.map → -->
      <li><a href="text/chapter1.xhtml">1장. 시작하며</a></li>
    </ol>
  </nav>
</body>
</html>
```

### ⑤ OEBPS/text/chapter[N].xhtml

- `convertToXhtml(chapter)`로 전체 XHTML 문서 생성
- XML 선언 + 올바른 namespace + `<link rel="stylesheet" href="../styles/style.css"/>`
- body 안에 `<h1 class="chapter-title">` (장 제목) + Tiptap HTML (XHTML 호환 변환 후)
- 본문 `<img src>`: data URL → `../images/img-NNN.ext` 로 치환 후 manifest 등록

### ⑥ OEBPS/styles/style.css

- `lib/epub/themes.ts`의 `readerThemeId` CSS 사용 (기존 4테마 이관)
- 공통: `line-height: 1.8`, 문단 여백, `.chapter-title`, `.epub-quote`, `.info-box`, `.drop-cap` 등

### XHTML 변환 (`lib/epub/convertToXhtml.ts`)

- `DOMParser`로 HTML 파싱 후 void 태그(`img`, `br`, `hr`) self-closing 보정
- 금지 태그 제거: `script`, `iframe`, `object`, `embed`
- `&` 등 XML 엔티티 이스케이프
- 정규식만으로 처리하지 말 것

### 이미지 처리

1. 에디터 업로드 → `FileReader` → data URL을 chapter content에 임시 저장
2. 패키징 시 data URL을 디코딩해 `OEBPS/images/`에 바이너리 파일로 추가
3. 본문·표지 각각 manifest 등록, 본문 src 경로 일괄 치환

## 6. AI 바이브 코딩 — 단계별 프롬프트

한 번에 전부 요청하지 말고, **완료 기준**을 확인한 뒤 다음 단계로 진행하세요.

---

### [1단계] 타입·훅·기존 page.tsx 골격 교체

```
이 프로젝트는 기존 PubForge XHTML 코드 에디터(app/page.tsx)를 WYSIWYG ePub 에디터로 전면 리팩터링한다.
lib/types/book.ts에 BookState/Chapter/BookMetadata/ReaderThemeId 타입을 정의하고,
hooks/useBookState.ts에서 useState + localStorage(epub_editor_book_state) 자동 저장/복구를 구현해줘.
app/page.tsx는 3단 레이아웃 헤더+빈 패널 골격만 남기고 기존 FileNode/textarea/export 로직은 제거해줘.
```

**완료 기준:** `npm run dev` 실행 시 빈 3단 레이아웃, 새로고침 후에도 BookState 유지.

---

### [2단계] 사이드바 — 메타데이터·장 관리·DnD

```
ChapterSidebar 컴포넌트를 만들어줘.
- 책 제목/저자/출판사/소개/표지 업로드 폼
- @dnd-kit으로 장 목록 드래그 순서 변경
- 장 추가/삭제/제목 인라인 편집/선택
- ThemePicker: lib/epub/themes.ts에 기존 4종 읽기 테마 CSS 이관
```

**완료 기준:** 장 CRUD·순서 변경·테마 선택이 State에 반영됨.

---

### [3단계] Tiptap WYSIWYG 에디터

```
@tiptap/react와 starter-kit, image, text-align, link, placeholder, underline을 설치하고
TiptapEditor 컴포넌트를 만들어줘.
툴바: B/I/U, H2/H3, 정렬, 링크, 이미지(FileReader data URL), 인용구, 구분선.
블록 삽입: 드롭캡 문단, info-box aside (기존 EPUB_TEMPLATES 콘텐츠 참고).
선택된 장의 content와 onUpdate 양방향 바인딩. script/iframe/style 태그 차단.
```

**완료 기준:** 장 전환 시 에디터 내용 교체, 타이핑 즉시 State·localStorage 반영.

---

### [4단계] 실시간 미리보기 (스마트폰 프레임)

```
LivePreview 컴포넌트: 스마트폰 프레임 UI + iframe srcDoc.
선택된 장의 HTML + readerThemeId CSS를 합쳐 렌더링.
Tiptap content 변경 시 실시간 갱신. 이전/다음 장 버튼(선택).
```

**완료 기준:** 에디터 입력이 1초 이내 미리보기에 반영, 테마 변경 시 스타일 즉시 변경.

---

### [5단계] XHTML 변환 유틸

```
lib/epub/convertToXhtml.ts를 작성해줘.
Tiptap HTML을 EPUB 본문에 넣을 XHTML 조각으로 변환 (void 태그, 엔티티 이스케이프, 금지 태그 제거).
generateChapter.ts에서 전체 chapter.xhtml 문서(선언문, head, stylesheet link, h1.chapter-title + body)를 생성.
```

**완료 기준:** 잘못된 `<img>`·`<br>` 포함 HTML도 유효한 XHTML 문서 생성.

---

### [6단계] 이미지 추출·manifest 연동

```
본문·표지의 data URL 이미지를 패키징 시 OEBPS/images/ 바이너리로 분리하고
src를 ../images/... 로 치환하는 extractImages() 유틸을 lib/epub/에 추가해줘.
```

**완료 기준:** 이미지 포함 장 다운로드 시 ZIP 내 images/ 폴더에 파일 존재.

---

### [7단계] ePub 패키징 (JSZip)

```
lib/epub/buildEpub.ts, generateOpf.ts, generateNav.ts를 구현해줘.
mimetype은 compression:'STORE'.
폴더 구조는 명세서 4장과 동일 (OEBPS/text/, OEBPS/styles/, OEBPS/nav.xhtml).
content.opf에 dc:identifier urn:uuid, dcterms:modified, nav properties, cover-image 포함.
헤더 [.epub 다운로드] 버튼 연결, 파일명 [책제목].epub, 다운로드 전 검증(5장) 적용.
```

**완료 기준:** 생성된 .epub을 Calibre·Apple Books 등에서 열리고 목차·본문·스타일 정상 표시.
